# 題庫專案工作規則

> 本檔為規則正本（Codex 讀這份；Claude Code 讀 CLAUDE.md，內容相同）。
> 接手工作前，請再讀：`交接文件.md`（架構與進度）、`docs/圖文混合題庫解析規範.md`（解析細則）。

## 鐵律（不可變動）

1. **提供給你的題庫來源檔與答案，一律不可修改**：不改寫、不修正、不推測、不補完。
   來源檔目前是大考中心官方 PDF；未來可能是課本、月考考卷、模擬考考卷等（PDF 或照片），規則相同。
2. **答案一律取自來源答案檔**，完成後逐題核對，零誤差才算完成。
3. **內容有疑義**（模糊、抽不出、版面錯亂）→ 標記人工複核（review_flags），不可默默修字或猜測。
4. **手寫題（非選擇題、作文）：放進題庫、列出題目，但不提供作答與判分**，同學可不用回答。
   開發過程中遇到手寫題，一律回報開發者：哪幾題是手寫題、共幾題。

## 題庫預設製作方式：「文字題＋必要裁圖＋題組共用資料」

- 純文字題盡量文字化（題幹、選項直接結構化）。
- 圖表、公式、插圖、無法可靠重建的版面（如雙欄），才裁來源檔局部圖。
- 題組的文章／表格／圖片只在題組層放一份，各題只存自己的題號、題幹、選項。
- 題目與答案完全依來源檔，不改、不猜。

## 答題卡標準格式（2026-07-13 起）

- 所有答題卡預設比照 115 國文重做後的方式：使用 `blocks/questions` 題庫結構，依正式卷順序呈現。
- 單題只放自己的題號、題幹、選項、答案；題組共用文章／表格／圖片只放在題組層一份。
- 可文字化的題幹與選項一律文字化；必要的圖表、公式、插圖或無法可靠重建版面，才裁官方局部圖。
- 手寫題放入同一份題庫並標示 `type: "handwritten"`，只列題目，不提供作答與判分。
- 不再新增獨立答題頁、測試頁、整份試卷頁面截圖版，除非先列出困難點並由傑克確認。
- 數學或公式密集題若需要裁圖，仍須遵守上述答題卡結構；若無法拆成合理互動題，先列 review_flags 給傑克確認。

## 工作慣例

- **唯一主程式** `public/index.html`：題庫一律做成 `src/question-bank/{年}-{科}.js`（`QUESTION_BANKS.push` 註冊）並掛進主程式；不可另開獨立頁。
- **不建模擬題／測試分類**進主程式（2026-07-10 已全數清除，不再出現）。
- 每批做完先完成**開發驗收**（渲染、判分、手機 360px），回報：題數／題組數／裁圖數／手寫題清單／待複核清單。
- **發佈方式**：開發驗收通過後**直接 `git push` 發佈上線**，只提供線上版給傑克檢查；不得再交付或標示其他離線／開發用版本。
- 來源檔格式優先順序（抽取可靠度由高到低）：**Word／純文字 ＞ 原生 PDF ＞ 掃描 PDF ＞ 照片**。
- 來源原檔不進公開 repo（`.gitignore` 已排除 `*.pdf` 等，技術上持續擋住——傑克 2026-07-10 確認維持）。
- 專案資料夾只放：程式、題庫、開發過程紀錄、交接文件（＋不公開來源原檔）。
