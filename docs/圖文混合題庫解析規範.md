# 圖文混合題庫解析規範

更新日期：2026-07-09

本文件記錄 115 年國綜小範圍測試後確立的方法。後續國文、英文、數學、社會、自然題庫均以此為共同開發基準。

## 一、不可變更的原則

1. 官方 PDF 是題目與答案的唯一依據。
2. 不可改寫、修正、推測或自行補完題目與答案。
3. OCR 只作位置與版面輔助，不可直接當正式題目內容。
4. 正式提取結果有疑義時，標記人工檢查，不可默默修字。
5. 國寫作文不納入互動作答。

## 二、解析優先順序

1. 使用 `pdfplumber` 提取文字、文字座標與表格。
2. 純文字題轉成結構化題幹與 A-D 選項，不產生圖片。
3. 表格優先使用 `extract_tables()`：
   - 可可靠辨識時，輸出結構化資料與 HTML。
   - 依官方版面建立 `rowspan`、`colspan`。
   - 不可用重複文字或空白格假裝合併儲存格。
4. 使用 PyMuPDF 提取真正的插圖或示意圖：
   - 只保存必要圖片本身。
   - 不裁入周圍可文字化的題幹與選項。
   - 圖片檔名包含批次或課次與題號。
5. 表格或複雜版面無法可靠重建時，才裁切官方 PDF 的必要區域。
6. OpenOCR 只協助辨識題號、表格、圖片及區域邊界。

## 三、題組判斷

解析下列標記：

- `XX-XX為題組`
- `閱讀下表，回答XX-XX題`
- `閱讀下圖，回答XX-XX題`
- 其他明確指定共用材料與題號範圍的文字

辨識為題組後：

1. 建立唯一 `題組ID`。
2. 共用表格、圖片或文章只保存一份。
3. 題組內各題只保存自己的題號、題幹與選項。
4. 不可把同一共用資料重複寫入每一道題目。

建議結構：

```json
{
  "題組": [
    {
      "題組ID": "L03-Q14-Q15",
      "課次": "L03",
      "題號範圍": [14, 15],
      "共用資料": {
        "類型": "table",
        "內容": {}
      },
      "題目": [
        {
          "題號": 14,
          "題幹文字": "...",
          "選項": {"A": "...", "B": "...", "C": "...", "D": "..."}
        }
      ]
    }
  ],
  "單題": []
}
```

## 四、表格重建

115 國綜第 14-15 題量詞表的確認結果：

- 表頭「量詞的類別」使用 `colspan="3"`。
- 「名量詞」外層跨完整名量詞區，共 6 個資料列。
- 「專用量詞」使用 `rowspan="5"`。
- 「借用量詞」的分類與細類使用 `colspan="2"`。
- 「動量詞」使用 `rowspan="2"`。
- 表格 JSON 可另保留展平資料供搜尋與分析，但前端必須使用合併版 HTML 還原官方結構。

## 五、前端題組呈現

1. 共用表格或圖片顯示在題組畫面上方。
2. 當前題目、選項與上一題／下一題控制顯示在下方。
3. 使用上下分割或 `position: sticky`，切換題目和捲動作答區時，共用資料仍保持可見。
4. 題目切換不得重新載入或複製共用資料。
5. 已選答案在題組內切換後應保留。

## 六、手機畫面要求

最終成品必須同時支援桌面與手機：

- 手機直向最低以 360px 寬度驗收。
- 另檢查手機橫向畫面。
- 寬表格放在共用資料區內水平捲動，不可造成整頁橫向溢出。
- 共用資料區高度必須受控，不可遮住整個題目與答題區。
- 題目區獨立垂直捲動時，上方共用資料仍可辨識。
- 選項、上一題、下一題與提交按鈕必須適合觸控。
- 所有文字、圖片、表格與控制不得重疊或被截斷。

## 七、每批題庫驗收

1. 題號、題幹、選項與答案逐題對照官方 PDF。
2. 確認純文字題沒有多餘圖片。
3. 確認表格內容、列欄與合併儲存格。
4. 確認插圖沒有裁入多餘文字，也沒有缺角。
5. 確認題組共用資料只保存一次。
6. 在桌面、手機直向、手機橫向操作題目切換與作答。
7. 使用者檢查通過前，不併入正式題庫，不公開發佈。
