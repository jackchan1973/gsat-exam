# 用 AI（Claude Code / Codex）打造「歷屆考題互動刷題工具」
## 開發參考 · 提示詞 · 完整流程

> 這份文件給**老師和家長**：你不用會寫程式。
> 只要有官方試題 **PDF** 或試卷**照片/圖片**，把下面的提示詞照抄丟給 **Claude Code** 或 **Codex**，
> 就能做出一個可以「刷題、自動對答案、看錯題」的網頁工具。
>
> 本文件依實際完成的高三學測歷屆題刷題工具整理而成。
>
> 注意：本文件是給老師與家長參考的通用教學稿；本專案實際開發規則仍以 `../AGENTS.md`／`../CLAUDE.md` 為準。

---

## 〇、先看這裡：三條鐵則（一定要讓 AI 遵守）

做題庫最怕 AI「自己編題目、自己想答案」。請務必把下面三條，連同你的提示詞一起交代給 AI：

1. **官方原檔是唯一依據**：題目、選項、文章、答案，只能照官方 PDF／圖片原樣，**禁止 AI 改寫、修正、推測、補完**任何內容。
2. **答案以官方答案檔為準**：做完要**逐題核對官方答案**，不可用 AI 自己算的答案。
3. **看不清楚就標記，不可硬掰**：AI 對某題不確定、或圖片模糊、或版面複雜抽不出來時，要**列出來讓人檢查**，不可自己猜一個填上去。

> 另外：**作文、國寫、非選擇題不做成互動作答**（無法自動判分），只保留官方 PDF 給學生看。

---

## 一、你需要準備什麼

- **官方試題 PDF**（最好同時有「試題」和「答案」兩個檔）——最推薦，效果最好。
- **或**：試卷的**清晰照片／掃描圖片**（每頁一張，字要清楚）。
- 一台電腦，已安裝 **Claude Code** 或 **Codex**。
- （可選）Python 環境。若用 PDF，AI 會用到 `pdfplumber`、`PyMuPDF` 這兩個工具，AI 會自己安裝，你不用管。

> ⚠️ **重點：大部分老師手上只有 PDF 或圖片。** 本文件的提示詞就是為「只有 PDF／圖片」設計的，AI 會從這些原檔一步步做出題庫，不需要你另外打字輸入題目。

---

## 二、整體流程（4 步，AI 幫你做，你負責檢查）

```
第1步  準備資料 ── 把官方 PDF 或圖片放進一個資料夾
          │
第2步  AI 讀原檔 ── 把 PDF/圖片整理成「題庫資料」（純文字＋表格＋必要的圖）
          │        （這步只忠實搬運，不做判斷，省 AI 用量）
          │
第3步  AI 做程式 ── 產生刷題網頁，並把題庫接進去
          │
第4步  人工驗收 ── 你拿官方 PDF 逐題對，確認沒錯才算完成
```

每一批（一科一年）做完都要**你親自對照官方檔驗收通過**，才進行下一批。**驗收通過前不要公開發佈。**

## 三、提示詞 1 ─ 製作「刷題學習程式」（前端 App）

> 用途：先做出一個**空的刷題程式外殼**（介面、答題卡、對答案、錯題、家長/老師檢查頁）。
> 題庫之後用「提示詞 2」再一批批加進去。
> 使用方式：新建一個空資料夾，在裡面開 Claude Code / Codex，把下面整段貼進去。

```
請幫我做一個「歷屆考題互動刷題」的網頁工具，先做程式外殼，題庫我之後再一批批加進來。

【技術要求】
- 純前端：HTML + CSS + JavaScript，不要後端、不要資料庫、不要框架，能直接用瀏覽器開。
- 手機優先：必須在手機直向 360px 寬度下正常顯示，文字、按鈕、圖片不重疊、不橫向溢出。
- 中文介面（繁體）。

【畫面流程】
科目 → 年度 → 分類 → 答題卡練習 → 交卷對答案 → 老師/家長檢查頁
- 首頁列出各科目（國文、英文、數學、社會、自然…），每科顯示有幾個年度、幾份官方 PDF。
- 選科目後選年度，選年度後選分類（例如「答題卡練習」），沒有題庫的年度就只顯示官方 PDF 連結可瀏覽。
- 進入練習頁是「答題卡模式」：整份題目一次列出，學生作答，最後按「交卷對答案」。

【題庫資料格式】
- 每一個題庫是一個 JavaScript 檔，用 `QUESTION_BANKS.push({...})` 把自己加進全域清單，
  這樣每加一科一年只要多一個檔、在 index.html 多掛一行 <script>，不會互相干擾。
- 一個題庫含：科目、年度、分類、標題、來源檔名、questions（全部題目的扁平清單，供判分）、
  blocks（呈現用的分段結構）。
- 題目欄位：題號 no、類型 type（single 單選 / multiple 多選）、題幹 stem、
  選項 choices（["A","B","C","D"]）、選項文字 options、正確答案 answer。

【必須支援的題型呈現】
1. 單選題：題幹 + A/B/C/D 圈選。
2. 多選題：可複選（checkbox）。
3. 題組（多題共用一篇文章/表格/圖）：共用材料顯示在上方並「固定可見」（sticky），
   下方切換題目時共用材料不重新載入、不重複、始終看得到。
4. 克漏字/文意選填：文章裡的空格用醒目底線標出題號；選項可以是「全題組共用的字庫/句庫」。
5. 圖片當選項的看圖題：四個選項是圖片。
6. 共用材料是一整張圖（例如複雜表格或雙欄文章）時，直接顯示該圖。

【判分與紀錄】
- 按「交卷對答案」後：自動比對官方答案，顯示得分（例：47/47）、列出錯題與正確答案。
- 多選題要完全選對才算對。
- 作答紀錄存在瀏覽器 localStorage；提供「老師/家長檢查」頁，顯示完成率、各分類答對/錯題數。
- 提供「重作」「清除紀錄」。

【重要原則】
- 作文/國寫/非選擇題不納入互動作答，只在該年度保留官方 PDF 連結可瀏覽。
- 先用 1~2 題清楚標示的「測試模擬題」把流程跑通即可，真正題庫我之後用另一份提示詞加。

做好後請告訴我怎麼用瀏覽器打開檢查（含手機 360px 怎麼測）。
```

## 四、提示詞 2 ─ 從「PDF 或圖片」製作題庫（**核心**）

> 用途：把某一科某一年的官方原檔，做成正式題庫，並接進上面的程式。
> **一次只做一科一年**，做完驗收再做下一批。

### 4A. 如果你有官方 PDF（最推薦）

```
我要把某一科某一年的官方學測/考試題，做成互動題庫，接進現有的刷題程式。

【資料位置】
官方 PDF 放在：<把你的資料夾路徑貼這裡>
裡面通常有：試題內容、答案（選擇題答案）、答題卷、非選擇題評分原則。

【務必遵守（不可違反）】
1. 官方 PDF 是題目與答案的唯一依據，不可改寫、修正、推測或補完任何內容。
2. 純文字題直接結構化，不要產生圖片。
3. 表格用程式抽取後，用 rowspan/colspan 正確還原合併儲存格，不可用重複文字或空白格假裝合併。
4. 只有「真正的插圖/示意圖/看圖選項」才裁切成小圖；能用文字表達的題幹和選項不可裁成圖。
5. 表格或版面太複雜、文字抽取會錯亂（例如雙欄）時，才退回「裁切官方 PDF 的那塊區域成一張圖」。
6. 題組（例：某幾題共用一篇文章/表格/圖）要判斷出來：共用材料只存一份，題組內各題只存自己的題號、題幹、選項。
7. 答案一律取自「官方答案 PDF」，做完要逐題核對，全部符合才算完成。
8. 有任何看不清楚或疑似抽取錯誤的地方，列出來讓我人工檢查，不可自行猜測修改。
9. 作文/國寫/非選擇題不納入互動作答，列成清單標明「不納入」，只保留 PDF 瀏覽。

【建議做法】
- 第一步先寫個小腳本，用 pdfplumber 抽「純文字（分頁）＋表格」、用 PyMuPDF 抽「圖片座標」，
  存成輕量的中繼 JSON。之後的判斷都根據中繼檔進行，不要每次重讀 PDF，省用量。
- 第二步再根據中繼檔，判斷題型（單題/題組/看圖/表格），組成正式題庫 JSON，
  並轉成程式用的題庫檔（用 QUESTION_BANKS.push 掛進 index.html）。
- 需要裁的圖，輸出成小圖片檔，檔名帶題號。

【完成後請回報】
- 這一科這一年總共幾題、幾個題組、幾張裁圖。
- 有沒有需要我人工複核的題目（附原因）。
- 怎麼在瀏覽器打開驗收（含手機 360px）。
- 每一題答案是否都與官方答案 PDF 相符（給我核對結果）。
```

### 4B. 如果你只有照片／圖片（沒有 PDF）

```
我只有試卷的照片/掃描圖片（沒有 PDF），要做成互動題庫，接進現有的刷題程式。

【資料位置】
圖片放在：<把你的資料夾路徑貼這裡>（每頁一張，已盡量拍清楚）
另外附上官方答案的圖片/清單：<貼上答案來源>

【務必遵守（最重要）】
- 這些圖片是官方試卷，請「看圖讀出」題目與選項，但這是唯一依據，
  你只能忠實抄錄，絕對不可改寫、潤飾、猜測或補完。
- 圖片模糊、看不清、不確定的字或符號，**一律標記「待人工確認」，不可自己填一個**。
- 答案以我提供的官方答案為準，逐題核對。
- 看圖選擇題、圖表題：需要的圖直接從原圖裁出來，不要用文字描述取代圖。
- 作文/非選擇題不納入互動作答。

【做法】
- 先把每張圖讀成「題號＋題幹＋選項」的結構，逐題列給我先確認，
  特別把你「不確定的字詞」用【?】標出來。
- 我確認後，再組成題庫並接進程式。

【完成後回報】
- 幾題、幾個題組、幾張需要用到的圖。
- 所有標【?】待確認的清單。
- 每題答案與官方核對結果。
- 怎麼在瀏覽器驗收（含手機）。
```

> 📌 **圖片來源的特別提醒**：從照片辨識文字（OCR）比 PDF 容易出錯。
> 用圖片時，一定要求 AI「逐題列出讓你先確認、不確定的用【?】標出」，你核對後再讓它組題庫。
> 有條件的話，先把圖片轉成 PDF 再用 4A 流程，通常更準。

## 六、常見狀況與處理（做題庫會遇到的）

| 狀況 | 正確處理 |
|------|----------|
| **題組**（幾題共用一篇文章/表格/圖） | 共用材料只存一份，放「題組層」；每題只存自己的題號、題幹、選項。畫面上共用材料固定顯示。 |
| **看圖選擇題**（選項是圖） | 把四個選項圖各裁一張；答案卡用 A/B/C/D 對應。 |
| **表格題** | 優先用程式抽表格，正確做出合併儲存格（rowspan/colspan）；不可用空白格假裝。 |
| **雙欄排版**（左右兩欄文章） | 文字抽取常會左右交錯錯亂。這種情況**直接裁那塊區域成一張圖**當共用材料，並標記人工複核。 |
| **多選題** | 類型設 multiple；要全部選對才算對。 |
| **作文 / 非選擇題** | 不做互動作答，列成「不納入」清單，只保留官方 PDF 瀏覽。 |
| **同一份不同年度長得不一樣** | 很正常。先請 AI 說明「今年跟去年哪裡不同、怎麼處理」，你同意再做。 |

## 七、驗收清單（你拿官方 PDF 逐項打勾）

- [ ] 題號、題幹、選項，逐題和官方 PDF 一字不差。
- [ ] 每題答案都和「官方答案」相符（請 AI 附核對結果）。
- [ ] 純文字題沒有多餘的圖片。
- [ ] 表格的欄列、合併儲存格正確。
- [ ] 插圖沒有裁到多餘文字，也沒有缺角。
- [ ] 題組的共用文章/表格/圖只出現一份。
- [ ] 桌機、手機直向（360px）、手機橫向都操作過：不重疊、不溢出、按鈕好按。
- [ ] 作文/非選擇題確實沒被放進互動作答。
- [ ] 標記「待人工確認」的項目都已一一確認。

> ✅ 全部打勾，這一批才算「正式完成」。沒全打勾前，不要公開分享給學生使用。

## 八、給老師的操作步驟（用 Claude Code 或 Codex）

1. 準備一個資料夾，把官方 PDF（或圖片）放進去，最好照「年度／科目」分子資料夾。
2. 在該資料夾開啟 **Claude Code** 或 **Codex**。
3. 先貼**提示詞 1**，做出刷題程式外殼，照它的指示用瀏覽器打開看看。
4. 再貼**提示詞 2（4A 或 4B）**，一次做一科一年題庫。
5. 做完**自己拿官方 PDF 對照第七節的驗收清單**，逐項打勾。
6. 驗收通過後，再貼**提示詞 3**做下一批。
7. 全部完成、確認無誤，才分享給學生使用。

> 💡 小訣竅：AI 一次做太多容易出錯也難檢查。**一次一科一年、做完就驗收**，最穩。

## 九、技術附註（給比較懂電腦的人參考）

- **架構**：純靜態網頁。`public/index.html` 掛載各題庫 JS（`src/question-bank/{年}-{科}.js`）與主程式 `src/app.js`、樣式 `src/style.css`。題庫用 `QUESTION_BANKS.push(...)` 各自註冊，互不干擾。
- **PDF 抽取**：`pdfplumber` 抽純文字與表格（`extract_tables`）、`PyMuPDF (fitz)` 抽圖片座標與裁圖。先產「中繼檔」（分頁純文字＋表格＋圖片座標）再做判斷，避免重複讀 PDF 消耗大量 token。
- **題庫產出**：建議寫一支可帶「年度參數」的建置腳本，把各年度差異（看圖題題號、裁圖座標、特殊題型）集中成設定，共用同一套解析流程。
- **判分**：選擇題比對官方答案自動判分（多選需完全一致）；進度存 `localStorage`。
- **驗證**：每次產出後，用程式把「每題答案 vs 官方答案 PDF」逐題比對，全對才算通過。
- 更詳細的圖文分離、題組、表格合併、手機驗收規則，可參考本專案 `docs/圖文混合題庫解析規範.md`。
