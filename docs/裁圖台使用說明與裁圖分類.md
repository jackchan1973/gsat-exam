# 人機協同裁圖台：使用說明與裁圖分類原則

建立日期：2026-07-23

## 為什麼有這個工具

以前裁圖是 AI「盲估」像素座標寫死在腳本裡（例如 `build_115_chinese_pymupdf_crops.py` 的 `box`），
裁完才知道對不對，容易切歪、缺角、破圖。改成**人機協同**：由開發者看著原頁**拖框**決定裁哪塊，
AI 負責其餘（渲染、換算、還原成 PNG、掛題庫）。這是「圖片題判分／裁圖不準」問題的治本做法。

定位：**開發期本機製作工具，不是上線產品**。試卷頁面圖屬來源內容，不進公開 repo。

## 檔案清單

| 檔案 | 角色 | 是否進 repo |
|---|---|---|
| `tools/裁圖台/index.html` | 三欄裁圖工具本體（通用，不含試卷內容） | ✅ 可提交 |
| `tools/build_crop_studio.py` | 資料產生器：渲染頁面圖＋整理中繼檔＋自動判規則頁 → `data.js` | ✅ 可提交 |
| `tools/apply_crop_coords.py` | 把裁圖台匯出的座標 JSON 精準還原成 PNG | ✅ 可提交 |
| `開啟裁圖台.command` | 雙擊啟動（起本機伺服器＋開瀏覽器） | ✅ 可提交 |
| `tools/裁圖台/data.js`、`pages/`、`thumbs/` | 每次換考卷產生的完整試卷頁面資料 | ❌ 已於 `.gitignore` 排除 |

## 使用流程

1. **AI 跑產生器**（換考卷改參數即可）：
   ```
   python tools/build_crop_studio.py \
     --pdf "資料/大考中心官方PDF/{年}學年度/{科}/…試題內容….pdf" \
     --extract "data/raw-extract/{年}學年度/{科}/…試題內容….json" \
     --year 115 --subject 國文 --dpi 300
   ```
   （來源 PDF 若只在主 repo，腳本會自動往上層找。）
2. **開發者雙擊 `開啟裁圖台.command`** → 瀏覽器開啟三欄工具：
   - **左＝原件**：縮圖選頁（規則頁自動標「跳過」）、頁面上拖框、方向鍵微調、可縮放。
   - **中＝解析結果**：核對抽取文字、比對原件「缺哪張圖就框哪張」。
   - **右＝裁圖工作區**：即時預覽、命名、加入清單、全部下載、匯出座標 JSON。
3. **開發者匯出**：下載 PNG ＋「匯出座標 JSON」，交給 AI。
4. **AI 還原並掛題庫**：
   ```
   python tools/apply_crop_coords.py --coords <座標.json> --out <輸出資料夾>
   ```
   再依 `blocks/questions` 結構組進 `src/question-bank/{年}-{科}.js`。答案一律取自答案檔。

### 幾個已內建的防呆
- **頁碼錯位**：左上同時標「PDF 第 N 頁 ＝ 試卷第 M 頁」（PDF 常比試卷多封面／規則頁）。
- **規則頁自動判定**：頁面文字沒有「第X頁／共Y頁」標記者，預設標記為跳過。
- **300dpi 高清**：裁出來的圖夠清楚。

## 裁圖 vs 文字化：判斷原則（重要）

「人機協同」的分工是：**開發者只框「真的是圖」的東西，純文字框交給 AI 文字化。**

| 類別 | 處理方式 | 例子 |
|---|---|---|
| 🟢 真圖／圖文混合／多欄版面 | **保留裁圖** | 插畫、書法、照片；文章嵌人物像；甲乙丙三欄 |
| 🟡 表格 | **理想重建 HTML 表格**，裁圖是退路 | 合併儲存格的量詞表 |
| 🔵 純文字（含框線） | **文字化**（框線用 CSS 畫） | 整段閱讀文章、單純文字框 |

為什麼純文字別裁圖：裁成圖後手機 360px 會縮成小條看不清；文字化可搜尋、可選取、會自動換行。

### 跨科通則：該文字就文字，該圖就圖（2026-07-23 定案，適用所有複習程式）

**這是所有複習程式、所有科目（國英數社自）的通則，不是數學專屬。** 判斷準則同一條：

- **能可靠文字化** → 文字（可搜尋、手機自動排版、好讀）。
- **不能可靠文字化** → 裁圖：圖形、公式、**特殊符號（分數／次方／根號／上下標）**、複雜版面（多欄）、圖文混合。
- **文字化前一律看原圖（裁圖或頁面圖）用視覺核對，特殊符號絕不猜**；看圖仍無法確定 → 改裁圖或標 `review_flags`。**絕不叫 AI 猜補／修復公式或符號**（違反鐵律不推測、不補完）。

各科「不能可靠文字化、要裁圖」的東西不同，但準則相同：

| 科 | 常見需裁圖 |
|---|---|
| 國文 | 書法、插畫、圖表、多欄版面、圖文混合、直排特殊符號 |
| 英文 | 地圖、圖表、看圖題、插圖 |
| 數學 | 幾何／函數圖、含分數／根號／次方的公式、圖形選項 |
| 社會 | 地圖、統計圖表、史料圖、漫畫 |
| 自然 | 實驗裝置圖、電路圖、化學結構式（上下標）、生物圖、數據圖表、公式 |

背景：`pdfplumber`／`PyMuPDF` 抽的是 PDF 底層文字流，分數／次方／上下標會錯位，硬文字化會出錯——這正是「文字化前看原圖核對」的原因。dry run（113 全國一模數學）實證：白話題幹／文字選項文字化漂亮；含分數／圖形選項的題整題裁圖。

**公式優先文字化（2026-07-23 起，已裝 KaTeX）**：答題卡已內建離線 KaTeX（`public/vendor/katex/`，免費、離線、含化學擴充 mhchem）。所以**公式優先用 LaTeX 文字表示**（題幹／選項以 `$...$` 包 LaTeX，如 `$\dfrac{2}{3}$`、`$x^2$`、`$H_2O$`），KaTeX 會渲染成課本般的公式——可搜尋、手機自動排版、忠實。做法：**看原圖把公式讀成 LaTeX → 渲染出來跟原圖對照 → 一致才用，讀不準才退回裁圖。** 只有「真正的圖」（幾何圖、函數圖、地圖、書法、插畫、複雜版面）才裁圖。

技術備忘：新題庫 JS 的 `<script>` 必須掛在 `115-chinese.js` **之後**（`QUESTION_BANKS` 在該檔第 2 行才定義），否則 push 會失敗。
