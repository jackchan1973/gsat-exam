# 開發流程

> 規則正本見 `CLAUDE.md`／`AGENTS.md`；架構與進度見 `交接文件.md`。本檔描述做一批新題庫的標準流程。

## 內容原則

- 提供的題庫來源檔（官方 PDF、課本、考卷掃描等）與答案，一律不可修改：不改寫、不修正、不推測、不補完。
- 題庫預設製作方式：**文字題＋必要裁圖＋題組共用資料**。
- 互動題庫採答題卡模式，收錄可自動判分的題型（單選、多選、選填數字格）。
- 非選擇題與作文暫不納入互動作答（方案待定）。
- 不建模擬題／測試分類進主程式。

## 一批題庫的標準流程

1. **確認來源檔**：放在 `資料/` 對應資料夾（官方 PDF 依 `{年度}學年度/{科目}/`；未來其他來源比照建資料夾）。
2. **產中繼檔**（若還沒有）：`tools/batch_extract.py` 抽純文字／表格／圖片座標到 `data/raw-extract/`。之後判斷盡量根據中繼檔，省 token。
3. **判斷科目型態**並回報：文字乾淨→文字化；公式密集（數學）→整題裁圖；混合→文字＋題組裁圖。照片來源→先逐題列出讓傑克確認，不確定的標【?】。
4. **寫／跑 build 腳本**：產出 `src/question-bank/{年}-{科}.js`（`QUESTION_BANKS.push`），需要的圖裁到 `public/images/{年}學年度/{科}/`。
5. **答案逐題核對**來源答案檔，零誤差才算數；疑義標 review_flags。
6. **掛進主程式**：`public/index.html` 加一行 `<script>`。
7. **本機驗收**：瀏覽器實測（渲染、判分、手機 360px），回報題數／題組數／裁圖數／待複核清單。
8. **等傑克確認**後才 `git push` 發佈（公開版不含任何來源原檔）。

## 製作順序

每一科先做 115 年那一份，再回補其他年度（見交接文件 §5 進度表）。

## 發佈原則

- 傑克完整檢查並確認之前，不進行公開發佈或正式部署。
- 公開版不含任何來源原檔；非官方來源（課本、模擬卷）之題庫要公開前須另行確認版權範圍。
