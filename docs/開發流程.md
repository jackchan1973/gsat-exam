# 開發流程

> 規則正本見 `CLAUDE.md`／`AGENTS.md`；架構與進度見 `交接文件.md`。本檔描述做一批新題庫的標準流程。

## 內容原則

- 提供的題庫來源檔（官方 PDF、課本、考卷掃描等）與答案，一律不可修改：不改寫、不修正、不推測、不補完。
- 題庫預設製作方式：**文字題＋必要裁圖＋題組共用資料**。
- 所有答題卡一律比照 115 國文重做後的方式：`blocks/questions` 結構、正式卷順序、題組共用資料只存一份、手寫題列出但不判分。
- 互動題庫採答題卡模式，收錄可自動判分的題型（單選、多選、選填數字格）。
- 手寫題（非選擇題、作文）：放進題庫、列出題目，但不提供作答與判分；開發時回報手寫題清單（第幾題、共幾題）。
- 不建模擬題／測試分類進主程式。

## 一批題庫的標準流程

1. **確認來源檔**：放在 `資料/` 對應資料夾（官方 PDF 依 `{年度}學年度/{科目}/`；未來其他來源比照建資料夾）。
2. **產中繼檔**（若還沒有）：`tools/batch_extract.py` 抽純文字／表格／圖片座標到 `data/raw-extract/`。之後判斷盡量根據中繼檔，省 token。
3. **判斷科目型態**並回報：文字乾淨→文字化；公式密集（數學）→必要公式/圖表局部裁圖；混合→文字＋題組裁圖。照片來源→先逐題列出讓開發者確認，不確定的標【?】。**同時回報手寫題清單（第幾題是手寫題、共幾題）**。
4. **寫／跑 build 腳本**：產出 `src/question-bank/{年}-{科}.js`（`QUESTION_BANKS.push`），需要的圖裁到 `public/images/{年}學年度/{科}/`；答題卡資料結構須使用 `blocks/questions`，不得新增獨立答題頁或整份試卷截圖版。
5. **答案逐題核對**來源答案檔，零誤差才算數；疑義標 review_flags。
6. **掛進主程式**：`public/index.html` 加一行 `<script>`。
7. **開發驗收**：瀏覽器實測（渲染、判分、手機 360px），回報題數／題組數／裁圖數／手寫題清單／待複核清單。
8. **直接 `git push` 發佈上線**，供開發者於線上檢查；發現問題再修正。

## 製作順序

每一科先做 115 年那一份，再回補其他年度（見交接文件 §5 進度表）。

## 發佈原則

- 開發驗收通過後直接發佈上線，只提供線上版給開發者檢查；不得再交付或標示其他離線／開發用版本（2026-07-13 起）。
