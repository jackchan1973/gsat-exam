# 開發流程

## 發佈原則

- 目前先以單機版開發與檢查為主。
- 在使用者完整檢查並確認之前，不進行公開發佈或正式部署。

## 內容原則

- 本專案是歷屆考題複習工具，題目與解答必須以官方 PDF 原檔為準。
- 開發過程不可改寫、修正、重製或自行更動任何題目與答案內容。
- 測試階段可以使用清楚標示的模擬題測試流程，但不可把模擬題標示成官方歷屆題。
- 互動題庫採答題卡模式，優先處理選擇題。
- 國寫作文題不納入程式作答；作文 PDF 僅作官方原檔瀏覽。
- 所有正式題目與答案仍以官方 PDF 為唯一依據，不可因解析方便而修改文字。
- 圖文分離、題組共用、表格合併與手機驗收的詳細規則見 [docs/圖文混合題庫解析規範.md](圖文混合題庫解析規範.md)。

## 目前流程

1. 從大考中心官方頁下載 PDF。
2. 依年度與科目整理到 `資料/大考中心官方PDF/`。
3. 用 `data/exam-files.json` 建立資料索引。
4. 用 `src/exam-files.js` 提供前端讀取。
5. 用 `public/index.html` 顯示篩選與開啟 PDF。

## 未來新增年度資料流程

1. 從大考中心官方頁下載該年度 PDF。
2. 放到：

```text
資料/大考中心官方PDF/{年度}學年度/{科目}/
```

3. 更新 `data/exam-files.json`。
4. 重新產生 `src/exam-files.js`。
5. 開啟 `public/index.html` 檢查是否能篩選與開啟。

## 題庫化流程

PDF 只是原始資料。要變成刷題工具，仍需要拆題。

建議優先順序：

1. 英文
2. 數學A / 數學B
3. 社會
4. 自然
5. 國綜 / 國寫

英文最適合先拆，因為題型結構清楚，且可直接做選擇題互動。
