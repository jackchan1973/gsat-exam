# 高三學測歷屆考題：交接文件

> 最後更新：2026-07-10
> 本專案由 **Claude Code 與 Codex 協同開發**：Codex 建立了最初的靜態網頁與 115 國文範例題庫；Claude Code 接手擴充答題卡互動題型、建立跨年度題庫產出流程、完成 113／114 英文題庫，並發佈上線。後續接手者（Claude Code 或 Codex）請先讀本文件與下方「不可變更原則」。

## 1. 專案目的

獨立的「高三學測歷屆考題」互動刷題工具，處理大考中心近三年（113/114/115）學測歷屆試題，跟 `vocab-app`（高一/高二段考複習）分開。定位是高三學測總複習：刷歷屆選擇題、自動對答案、錯題與弱點紀錄。

## 3. 唯一主程式與架構

**所有互動題庫都整合進單一主程式 `public/index.html`，不可另開獨立頁。**

```text
public/index.html            主程式（掛載各題庫 JS 與 app.js）
 ├─ src/exam-files.js        EXAM_FILES：官方檔案索引（提供科目/年度清單）
 ├─ src/mock-questions.js    MOCK_QUESTION_SETS：清楚標示的測試模擬題
 ├─ src/question-bank/*.js   各科各年題庫（QUESTION_BANKS.push(...) 各自註冊）
 ├─ src/app.js               SPA：科目→年度→分類→答題卡→師長檢查（進度存 localStorage）
 └─ src/style.css            樣式
index.html（根目錄）          自動導向 public/index.html（給 GitHub Pages 用）
```

SPA 流程：**選科目 → 選年度 → 選分類 → 答題卡練習 → 交卷自動判分 → 師長檢查（完成率／錯題／進度）**。

app.js 目前支援的題型呈現：純文字單選/多選、圖片題組（displayGroups，115 國文用）、以及英文的豐富題型（共用文章 sticky、字庫/句庫、克漏字空格、圖片選項、雙欄文章裁圖、論壇文字多選）。手機 360px 已驗收。

## 4. 題庫產出流程（範本已建立）

```text
官方 PDF ──(tools/batch_extract.py)──▶ 中繼檔 data/raw-extract/{年}學年度/{科}/*.json
   （抽純文字/表格/圖片座標，只忠實搬運，不做判斷，省 token）
           │
           ▼
中繼檔 ──(build 腳本)──▶ src/question-bank/{年}-{科}.js（QUESTION_BANKS.push），在 index.html 掛一行 <script>
           │
           ▼
自動出現在主程式，走答題卡流程；答案取自官方答案 PDF，逐題核對零誤差才算數。
```

- 英文用**跨年度單一腳本** `tools/build_english.py {年}`：各年度差異（看圖題題號與裁圖座標、混合題型態）只寫在腳本內 `CONFIG`，共用同一套解析。
- 需裁的圖（看圖選項、雙欄文章）輸出到 `public/images/{年}學年度/{科}/`。

## 6. 製作順序（2026-07-10 更新）

**改為「每一科都先做 115 年那一份」**，各科先各做一個，方便提早在不同科目型態上發現問題，之後再回頭補其他年度。

- 目標互動題庫共 18 份（6 科 × 3 年，國寫作文不納入）。
- 待做的 115：115 英文、115 數學A、115 數學B、115 社會、115 自然。（115 國文 Codex 已做；未來會用 build 流程重做成一致格式後，再刪掉 Codex 原版。）
- 各科差異：國文（文言文、題組多）、英文（結構最乾淨，混合題每年型態不同）、數學（大量公式/符號變小圖，多需裁圖）、社會/自然（超多圖表與題組，圖最多）。

## 7. 部署與「不放官方 PDF」規則

- GitHub Pages 從 `main` 分支發佈；更新＝`git push` 後約 1–2 分鐘生效。
- **公開版絕不含任何官方 PDF 原檔（版權）**：`.gitignore` 已排除 `資料/大考中心官方PDF/`、`*.pdf`；前端已移除「查看官方 PDF 原檔」瀏覽功能。
- 官方 PDF 只留在**本機**當建題庫的來源資料（`資料/大考中心官方PDF/`，81 個）。
- `.gitignore` 也排除 `.venv*/`、`tmp/`、`.claude/` 等，避免上傳。

## 8. 不可變更原則（務必遵守）

1. 官方 PDF 是題目與答案的唯一依據，不可改寫、修正、推測或補完任何內容。
2. 答案一律取自官方答案檔，做完逐題核對，全部相符才算完成。
3. 純文字題直接結構化不產生圖片；只有真正的插圖/看圖選項/無法可靠重建的版面才裁圖。
4. 題組共用資料（文章/表格/圖）只存一份於題組層。
5. 抽取有疑義（模糊、抽不出、版面錯亂）一律標 review_flags 交人工複核，不可自行猜測修改。
6. 國寫作文與非選擇題不納入互動作答。
7. 在使用者完整驗收確認前，不視為正式完成、不公開發佈；公開版不放官方 PDF。
